# Ciggies.app 复刻项目设计文档

> 项目目标：复刻 https://www.ciggies.app/ 网站，一个卷烟品牌/产品数据库与社区平台

---

## 一、项目概述

### 1.1 网站定位
Ciggies.app 是一个卷烟收藏与社区平台，提供：
- 卷烟产品数据库（品牌、规格、图片、价格、参数）
- 用户收藏功能（收藏、尝试过、愿望清单）
- 社区互动（排行榜、动态流、实时聊天）
- 多语言支持（中英双语）

### 1.2 目标用户
- 卷烟收藏爱好者
- 对卷烟文化感兴趣的用户
- 寻找特定品牌信息的消费者

---

## 二、技术栈规范

### 2.1 前端技术
- **框架**: React 18+
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **构建工具**: Vite
- **路由**: React Router v6
- **状态管理**: Zustand 或 React Context
- **HTTP 客户端**: Axios 或 Fetch

### 2.2 后端技术（推荐）
- **Node.js + Express** 或 **Next.js API Routes**
- **数据库**: PostgreSQL 或 MongoDB
- **ORM**: Prisma 或 Mongoose
- **认证**: JWT + OAuth (Google, GitHub)

## 四、数据模型设计

### 4.1 核心实体

```typescript
interface Product {
  id: number;

  packPrice: number;          // 单包价格 ¥

  cartonBarcode: string;

  ratingTaste: number;
  ratingPack: number;
  ratingValue: number;

  ratingVotes: number;

  updatedAt: Date;
```

## 五、API 接口设计

### 5.1 产品相关
```
GET    /api/products              # 获取产品列表（支持分类筛选、搜索）
GET    /api/products/:id          # 获取产品详情
GET    /api/brands                # 获取品牌列表
GET    /api/brands/:id            # 获取品牌详情（含该品牌下所有产品）
```

### 5.2 用户相关
```
POST   /api/auth/register         # 用户注册
POST   /api/auth/login            # 用户登录
POST   /api/auth/logout           # 用户登出
GET    /api/auth/me               # 获取当前用户信息
GET    /api/users/:username       # 获取用户公开信息
```

### 5.3 用户产品操作
```
POST   /api/user/products         # 添加收藏/尝试/愿望清单
DELETE /api/user/products/:id     # 移除收藏/尝试/愿望清单
GET    /api/user/products         # 获取用户的产品列表
```

### 5.4 社区相关
```
GET    /api/community/leaderboard # 获取排行榜
GET    /api/community/feed        # 获取动态流
```

### 5.5 聊天相关
```
GET    /api/chat/messages         # 获取历史消息
POST   /api/chat/messages         # 发送消息
WS     /ws/chat                   # WebSocket 实时聊天
```

## 六、开发规范

### 6.1 服务器管理
- **单一开发服务器规则**：每个项目同时只能运行一个开发服务器
- 启动前检查端口占用：`lsof -i :<port>` 或 `netstat -tulpn | grep <port>`
- 不要重复启动服务器实例

### 6.2 Git 工作流
- 使用 git worktree 进行功能分支开发
- 保持工作区整洁
- 频繁提交，提交信息有意义
- 遵循已有代码模式和架构

### 6.3 代码质量
- 遵循 TypeScript 严格模式
- 使用 ESLint + Prettier
- 组件命名采用 PascalCase
- 文件命名采用 kebab-case
- 保持命名一致性

### 6.4 资源管理
- 注意磁盘空间使用
- 定期清理临时文件
- 运行 `npm audit` 检查安全警告
- 不要提交敏感信息

## 七、UI/UX 规范

### 7.1 设计风格
- **整体风格**：简洁、现代、档案感
- **配色方案**：
  - 主背景：深色（#0f0f0f 或类似）
  - 卡片背景：略浅的深色
  - 强调色：品牌色（可自定义）
  - 文字：白色/灰色层级

### 7.2 响应式设计
- 移动端优先
- 产品卡片网格自适应
- 侧边聊天面板在移动端可收起

### 7.3 交互设计
- 产品卡片 hover 效果
- 按钮点击反馈
- 加载状态骨架屏
- 无限滚动或分页
